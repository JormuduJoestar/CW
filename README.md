# Telegram Channel Admin System (Hybrid Architecture)

Программная система для автоматизации администрирования Telegram-каналов. Решает проблемы рутинного контент-менеджмента (мульти-постинг, шаблонизация) и преодолевает ограничения официального Bot API в части сбора статистики просмотров.

## 🌟 Ключевые особенности

Проект построен на **гибридной двухуровневой архитектуре**:
1. **Контур Bot API (`aiogram`)**: Обеспечивает быстрый и отзывчивый интерфейс для администратора (FSM, inline-клавиатуры, управление отложенными задачами).
2. **Контур User API (`Pyrogram`)**: Изолированный фоновый модуль парсинга (MTProto). Авторизуется как обычный пользователь, «прогревает кэш» диалогов и собирает реальную статистику просмотров (views), недоступную для обычных ботов.

## 🛠 Технологический стек

* **Язык:** Python 3.10+
* **Интерфейс:** `aiogram 3.x`
* **Парсер:** `Pyrogram 2.x`
* **База данных:** `SQLite` + `SQLAlchemy 2.0` (асинхронный драйвер `aiosqlite`)
* **Планировщик:** `APScheduler`
* **Развертывание:** `Docker`, `Docker Compose`

## ⚙️ Основной функционал

* **Безопасность:** Автоматическая валидация прав администратора при привязке канала.
* **Шаблонизация:** Сохранение в БД текста и медиа-вложений для быстрого переиспользования.
* **Мульти-постинг:** Рассылка одного поста в N каналов одновременно.
* **Планирование:** Отложенная публикация по заданному времени.
* **Автоудаление:** Настройка таймера (TTL) для временных рекламных постов.
* **Статистика:** Сбор просмотров активных постов по запросу и автоматическое сохранение финальной статистики перед автоудалением поста.

## 🚀 Инструкция по локальному запуску

### 1. Подготовка окружения
Склонируйте репозиторий и установите зависимости:
```bash
git clone https://github.com/ВАШ_ЛОГИН/ВАШ_РЕПОЗИТОРИЙ.git
cd ВАШ_РЕПОЗИТОРИЙ
python -m venv venv
source venv/bin/activate  # Для Linux/macOS
pip install -r requirements.txt

2. Конфигурация

Создайте файл .env в корневой директории и заполните его:

BOT_TOKEN=123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11
ADMIN_ID=123456789
API_ID=1234567  # Получить на my.telegram.org
API_HASH=0123456789abcdef0123456789abcdef
SESSION_NAME=my_userbot_session

3. Инициализация сессии MTProto

Перед первым запуском необходимо создать файл сессии для парсера:

python init_session.py

Следуйте инструкциям в консоли: введите номер телефона аккаунта-помощника и код
подтверждения из Telegram.

4. Запуск системы

python main.py

При первом запуске автоматически создастся база данных bot_database.db и все
необходимые таблицы.

🐳 Развертывание через Docker

Для запуска в production-среде используйте Docker Compose. Убедитесь, что файлы
.env и *.session (сгенерированный на шаге 3) лежат в корневой директории.

docker-compose up -d --build

База данных сохраняется на хост-машине с использованием механизма Docker
Volumes.
